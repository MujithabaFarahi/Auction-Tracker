1. Purpose

Build a Minimum Viable Product (MVP) web application to track a live cricket player auction.

This MVP is NOT a full tournament management system.
It is ONLY for running and viewing a single auction for one tournament.

2. Technology Stack (Fixed)

Frontend: React (web)

Database: Firebase Firestore

Authentication: Firebase Auth (Admin only)

Real-time: Firestore real-time listeners (no WebSockets)

‚úÖ Only auction tracking

‚ùå No multi-tournament support

‚ùå No payments

‚ùå No analytics

‚ùå No chat, notifications, or extras

‚ùå No captain logins (view-only)

MVP UPDATE: Live Bidding History (MANDATORY FEATURE)
1. Requirement Summary

During a live auction, the system MUST display the full bidding sequence for the currently auctioned player.

This is a must-have feature, not optional.

Captains and admins must be able to see:

Who bid

How much was bid

In what order the bids happened

This improves transparency and trust in the auction.

2. Bidding Flow (Expected Behaviour)

Example scenario:

Player base price: 20,000

Team A bids: 20,000

Team B bids: 25,000

Team C bids: 30,000

Team A bids: 40,000

Team B bids: 50,000 (winning bid)

üëâ All of the above bids must appear live on the screen in sequence.

5. Auction Completion Rule (With Bid History)

When admin marks player as SOLD:

Winning bid = last entry in bid history

Player record updated:

status = SOLD

soldPrice = winning bid amount

soldToTeamId = winning team

Team purse deducted

Auction state reset:

currentBid = 0

leadingTeamId = null

bidHistory = []
